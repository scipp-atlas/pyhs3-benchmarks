Getting Started¶

This guide introduces the basic workflow for working with the PyHS3 Benchmarks repository.

By the end of this guide you will be able to

  • install the project;
  • execute benchmark suites;
  • understand the generated outputs;
  • generate benchmark plots;
  • navigate the repository documentation.

If you have not yet installed the project, see the Installation guide before continuing.

Benchmark Workflow¶

A typical benchmark session consists of four steps.

Install Dependencies
        │
        ▼
Run Benchmarks
        │
        ▼
Inspect Results
        │
        ▼
Generate Plots

The repository automates this workflow through the matrix runner and the plotting utilities.

Running Your First Benchmark¶

The recommended way to execute benchmarks is through the matrix runner.

For example,

pixi run python -m src.run_alexx_workspace_benchmark_matrix \
    --workspace-dir inputs/alexx_workspaces \
    --benchmarks workspace_loading

This command automatically

  • discovers benchmark workspaces;
  • executes the selected benchmark;
  • stores benchmark reports;
  • generates the standard output directory structure.

For a description of the available benchmark suites, see the Benchmark Suite documentation.

Running Multiple Benchmarks¶

Several benchmark suites can be executed during a single run.

pixi run python -m src.run_alexx_workspace_benchmark_matrix \
    --workspace-dir inputs/alexx_workspaces \
    --benchmarks \
        workspace_loading \
        model_creation \
        compiled_evaluation

To execute every supported benchmark,

pixi run python -m src.run_alexx_workspace_benchmark_matrix \
    --workspace-dir inputs/alexx_workspaces \
    --benchmarks all

Benchmark Outputs¶

Benchmark execution produces structured JSON reports stored in the results/ directory.

A typical output hierarchy looks like

results/

    workspace_loading/

        10ch_bkgGenExp_sigGeneric_shapeFloat_npOn_constrGauss_yield1x/

            repeat_000/

                workspace_loading_result.json

See Benchmark Results for a detailed description of the output format.

Generating Plots¶

Most benchmarks can optionally generate plots during execution.

pixi run python -m src.run_alexx_workspace_benchmark_matrix \
    --workspace-dir inputs/alexx_workspaces \
    --benchmarks workspace_loading \
    --plot

Generated figures are written to the plots/ directory.

Benchmark Overview¶

After executing one or more benchmark suites, an overview report can be generated.

pixi run python -m src.plot_benchmark_overview \
    --results-dir results/alexx_workspace_matrix \
    --plot-dir plots/alexx_workspace_matrix/benchmark_overview

The overview aggregates benchmark reports into publication-quality summary figures covering

  • execution time;
  • memory usage;
  • benchmark status.

Typical Development Workflow¶

The recommended workflow when developing or modifying benchmarks is

Modify Benchmark
        │
        ▼
Execute Matrix Runner
        │
        ▼
Inspect JSON Reports
        │
        ▼
Generate Overview Plots
        │
        ▼
Verify Results

Using the same workflow throughout the repository ensures that benchmark outputs remain consistent and directly comparable.
